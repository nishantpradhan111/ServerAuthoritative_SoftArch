Take-Home Evaluation Components: Submission and Evaluation
Guidelines for CS G653/ SS G653
Software Design and Development (team-based; team-size – 1 to 4)
Submission Guidelines
1. Abstract Submission
o

The abstract must clearly define the problem statement, objectives,
system scope, and key functionalities.

o

It should justify the relevance of the proposed system and its suitability
for architectural comparison.

o

The abstract is subject to instructor approval before implementation
begins.

2. Architectural Implementations
o

The software system must be fully implemented using one architectural
pattern.

o

The pattern can be chosen freely by a team. Certain constraints would be
specified after abstract submission (check the excel sheet)

3. Code and Documentation
o

Complete, well-structured, and executable source code must be
submitted for the implementation.

o

Comprehensive architectural documentation must be prepared for the
pattern using the techniques taught in the course (e.g., views, diagrams,
rationale, and quality attribute considerations). It should contain a link to
the git repository.

o

Clear instructions for setup, execution, and testing must be included.

4. Final Submission
o

All deliverables (code, documentation, and abstract) must be submitted
within the prescribed deadline in the specified format.
